# Multi-Threaded-Log-Analyzer
Multi-Threaded Log Analyzer

Project Overview
This project demonstrates the concept of Parallel Computing using Multi-Threading to analyze large log files efficiently. Log files generated by servers, applications, or systems can become extremely large and contain thousands or even millions of lines. Processing these files sequentially using a single thread can be slow and inefficient.

The main idea of this project is to improve the performance of log analysis by dividing the log file into smaller sections and processing them simultaneously using multiple threads. Each thread is responsible for analyzing a specific portion of the file, allowing the system to process large data sets faster than traditional sequential processing.

Problem Statement

Large log files are commonly generated in modern systems such as web servers, databases, and distributed applications. These logs contain important information about system events, errors, warnings, and user activities. However, analyzing these files using a single thread requires scanning the entire file line by line, which may take a long time when the file size is large.

To solve this problem, the project uses multi-threading, where several threads run in parallel. Each thread processes a different part of the log file at the same time, which significantly reduces the overall execution time.

Project Objectives

The main objectives of this project are to apply the concepts of parallel computing, improve the performance of log file analysis, and demonstrate how multi-threading can speed up large-scale data processing. The system also aims to detect and count different types of log messages such as ERROR, WARNING, and INFO.

How the System Works

The system follows several steps. First, the program reads a log file containing system messages. Then the file is divided into multiple smaller chunks. After that, multiple threads are created, where each thread processes one portion of the file. Every thread scans its assigned section and counts specific log levels such as ERROR, WARNING, and INFO. Finally, the results from all threads are combined to produce the final statistics.

Technologies Used

The project is implemented using Python, specifically the threading library to create and manage threads. The project is also managed and documented using GitHub.

Expected Output

After processing the log file, the program displays statistics showing the number of each log type.

Example:
ERROR: 25
WARNING: 13
INFO: 102

Parallel Computing Concept Used

The project applies the concept of multi-threading, where multiple threads run simultaneously to perform tasks in parallel. This reduces the total processing time compared to sequential execution.

Future Improvements

Future improvements may include analyzing IP addresses that generate the most errors, visualizing log statistics using graphs, comparing single-thread and multi-thread performance, and supporting extremely large log files.
